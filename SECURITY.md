# 安全说明

## 报告安全问题

请通过仓库所有者提供的私密渠道报告漏洞。不要在公开问题、日志或截图中提交密码、Token、Cookie、API Key、OpenVPN 私钥、完整配置文件、真实公网地址或可复现的生产凭据。

报告应包含受影响版本、最小复现条件、预期与实际行为以及已经脱敏的诊断信息。在修复和发布安全更新前，请避免公开利用细节。

## 支持边界

当前支持线为 `0.1.x`。系统默认使用模拟网络执行器；真实 Namespace、OpenVPN、防火墙、SOCKS5、扫描、IP 情报、解锁检测、连接生命周期和自动切换都需要独立的显式门禁。

VPNGate 节点由第三方志愿者提供，可能记录流量元数据、下线、限速、更换 IP 或被目标服务封锁。网络类型和住宅属性属于概率判断。使用者必须遵守当地法律、VPNGate 规则及目标服务条款，不得用本系统绕过法律或平台安全控制。

## 运行安全基线

- 只从校验通过的发布归档安装。
- 使用专用非 root 服务账户和项目提供的最小 sudoers 规则。
- 生产环境使用受信任 TLS 证书，限制管理页面来源并保护 root 配置。
- 定期执行 `sudo bash scripts/diagnose.sh`，监控审计日志和升级备份。
- 首次真实网络测试使用可丢弃虚拟机和带外控制台，不在唯一 SSH 管理链路上测试。
- 泄露凭据后立即轮换并撤销相关 Session；不要只删除日志。
